Team 9: Making problems vanish

The challenges we faced

The problem statement


  • A common platform where cities and their citizens can post problems in their region
  • The users and cities can share their solutions to the existing problems
  • Share, collaborate and adapt a working solution in any city

Our Approach


  • Identified the user flow and mapped them to entities
  • We drew out the intended site flow and began building
  • We discusseded needs from the database and mapped our database (slide later)
  • Then we started filling the pages with sample data

Challenges We Encountered


  • Scale of the project
  • Not having a server to develop as a team
  • There were issues with communication
    between front end back end
  • Dealing with multiple languages
    • Java (Back End)
    • MySQL (Database)
    • PHP (Front End Server Side)
    • Javascript (Front End Client Side)
    • Bootstrap (CSS and Formatting)

What We Accomplished

The front end side

  • Our website showcases our user's profile, allowing them to tag their skillsets and interests.
  • The user can comment on forums to discuss possible problems in their city and solutions to them.
  • The city utilizes these post by using the solutions or by giving these solution to other cities.

The back end side

  • Our database overview shows the flow of information from the indivual to the user.
  • The admins moniter the posts the users create, based on the city's ID.
  • The user's posts are time stamped so others can edit the and comment on the post.

Our Website Overview:

Implemented features


  • Local databases to build and store a user's profile.
  • The actual webpage with interactive dropdowns. Which is also mobile friendly.
  • User forums for collaration and discussion. (reddit Style)

Future implementable features



  • Server datebase scaling for our website.
  • More custom user features.
  • Downvote and upvote system for user ratings.
  • Text/Email notifications
  • Public/Private projects (currently all public)
